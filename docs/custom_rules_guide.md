# 自定义规则系统使用指南

## 核心功能

支持 KP/DM 定义私人玩法的自定义规则和属性，AI 自动理解并应用。

## 使用场景

### 场景 1: 自定义属性

```
KP: 我们的游戏有"灵力"属性
AI: [调用 define_attribute("灵力", "数值", "50", "角色的灵力值，用于施展法术")]

KP: 角色还有"因果点"
AI: [调用 define_attribute("因果点", "数值", "3", "可消耗改变命运的特殊点数")]
```

之后 AI 会自动：
- 在角色卡中包含这些属性
- 理解相关检定
- 在叙述中考虑这些属性

### 场景 2: 自定义规则

```
KP: 我们使用"优势叠加"规则：多个优势来源可以叠加
AI: [调用 define_rule("优势叠加", 
     "当角色同时获得多个优势时，每个额外优势+2加值",
     "地形优势+魔法增益=d20+2+2")]

KP: 失败可以燃烧因果点重投
AI: [调用 define_rule("因果重投",
     "检定失败后可消耗1因果点重新投骰，取较好结果",
     "侦察失败→消耗因果点→重投成功")]
```

### 场景 3: 复杂自定义系统

```
KP: 我们的魔法系统：
- 法术消耗灵力
- 灵力不足会受到反噬
- 反噬造成1d6伤害并获得"虚弱"状态

AI: [自动调用多个 define_rule 和 define_attribute]
✅ 已定义魔法系统规则
```

之后施法时：
```
玩家: 我施展火球术
AI: 火球术消耗15灵力。你当前灵力：42
    [自动计算] 剩余灵力：27
    
    你伸出双手，炽热的火球在掌心凝聚...
```

## 实际案例

### 案例 1: 末日废土设定

```
define_attribute("辐射抗性", "数值", "0", "对辐射的抵抗力")
define_attribute("变异点", "数值", "0", "累积的变异程度")

define_rule("辐射检定", 
  "进入辐射区每小时进行体质检定，失败获得1变异点",
  "辐射区3小时→3次检定→失败2次→获得2变异点")

define_rule("变异效果",
  "变异点达到5/10/15时触发随机变异",
  "10变异点→触发变异→获得夜视但畏光")
```

### 案例 2: 武侠设定

```
define_attribute("内力", "数值", "100", "角色的内力值")
define_attribute("轻功", "数值", "50", "轻功技能等级")

define_rule("内力运功",
  "消耗内力可以提升检定结果，1内力=+1加值",
  "攻击检定15→消耗5内力→最终20")

define_rule("轻功移动",
  "轻功检定成功可以无视地形移动",
  "轻功70→跨越10米鸿沟成功")
```

### 案例 3: 赛博朋克设定

```
define_attribute("义体等级", "数值", "0", "义体改造等级")
define_attribute("人性", "数值", "100", "剩余人性值")

define_rule("义体安装",
  "安装义体提升能力但降低人性",
  "安装战斗义肢→力量+5→人性-10")

define_rule("人性崩溃",
  "人性降至0时进入赛博精神病状态",
  "人性5→再次改造→人性0→触发精神病")
```

## AI 如何应用自定义规则

### 自动理解
```
玩家: 我用灵力强化这次攻击
AI: [检查自定义属性"灵力"]
    你消耗10点灵力，剑刃泛起青色光芒
    [自动计算] 灵力: 42→32
    攻击检定获得+5加值
```

### 智能裁定
```
玩家: 我能用因果点改变这次失败吗？
AI: [检查自定义规则"因果重投"]
    可以！你消耗1因果点，时间仿佛倒流...
    [重新投骰]
```

### 主动提醒
```
AI: 你已经在辐射区待了3小时，需要进行第3次体质检定
    [自动应用"辐射检定"规则]
    你感到一阵眩晕... [检定失败]
    你的变异点增加到3
```

## 最佳实践

1. **清晰描述** - 规则描述要具体明确
2. **提供示例** - 给出实际应用例子
3. **逐步定义** - 先定义基础，再添加复杂规则
4. **测试验证** - 定义后立即测试 AI 是否理解

## 命令速查

```python
define_attribute(名称, 类型, 默认值, 描述)
define_rule(规则名, 描述, 示例)
```

类型选项：数值、文本、布尔、列表
