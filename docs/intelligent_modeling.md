# 智能规则建模系统

## 核心理念

**AI 自动理解规则，而非手动定义**

传统方式：
```
KP: define_attribute("灵力", "数值", "100", "...")
KP: define_rule("施法消耗", "...", "...")
```

智能方式：
```
KP: [上传规则文档 rules.pdf]
AI: 我已理解你的规则系统：
    - 检测到属性：灵力、内力、因果点
    - 理解规则：施法消耗灵力、内力可强化攻击
    - 掌握检定：灵力检定、内力运功
```

## 工作流程

### 1. 上传规则文档

```
KP: 这是我们的自定义规则书 [上传 custom_rules.pdf]
AI: [自动分析文档]
    ✅ 已理解规则系统
    - 发现 8 个自定义属性
    - 识别 15 条游戏规则
    - 理解 5 种检定方式
```

### 2. AI 自动应用

**场景：玩家行动**
```
玩家: 我用内力震碎这扇门
AI: [自动思考]
    - 需要属性：力量 + 内力
    - 检定方式：力量检定
    - 规则应用：内力运功（消耗内力获得加值）
    
    [自动执行]
    你运转内力，真气汇聚于掌心...
    力量检定：d20+5（力量）+10（消耗10内力）= 23
    
    轰！木门应声而碎
    [内力：50→40]
```

**关键：AI 自己判断，不问玩家**

### 3. 智能属性选择

```
玩家: 我想说服守卫放我们进去
AI: [分析情境]
    - 社交场景
    - 需要：魅力 + 口才技能
    - 可选加成：如果有"贵族身份"背景
    
    [自动检定]
    你展示出优雅的谈吐...
    魅力检定：18（成功）
    守卫被你的气质打动，让开了道路
```

## 实际案例

### 案例 1：武侠设定

**规则文档内容：**
```
# 武侠规则

## 属性
- 内力：角色的内功修为（0-100）
- 轻功：轻身功夫等级（0-100）
- 剑法：剑术造诣（0-100）

## 规则
- 内力运功：消耗内力可提升检定，1内力=+1
- 轻功移动：轻功检定成功可无视地形
- 剑气：剑法70+且内力50+可发出剑气远程攻击
```

**游戏中：**
```
玩家: 我跃上屋顶追击
AI: [识别：需要轻功]
    你足尖点地，身形如燕...
    轻功检定：75（成功）
    你轻松跃上三丈高的屋顶

玩家: 我发出剑气攻击
AI: [检查：剑法72，内力60，满足条件]
    你剑指一引，一道青色剑气破空而出！
    [消耗10内力]
    攻击检定：d20+剑法加值
```

## 优势对比

### 传统手动定义
```
❌ 需要逐个定义属性
❌ 需要写规则描述
❌ AI 被动接收
❌ 容易遗漏
```

### 智能建模
```
✅ 上传文档自动理解
✅ AI 主动分析
✅ 智能选择属性
✅ 自然应用规则
```

## 使用建议

1. **规则文档要清晰**
   - 明确列出所有属性
   - 说明规则的触发条件
   - 提供应用示例

2. **让 AI 自主判断**
   - 不要问"用什么属性"
   - 信任 AI 的理解能力
   - 必要时可以纠正

3. **测试验证**
   - 上传后进行几次测试
   - 确认 AI 正确理解
   - 发现问题及时补充文档

## 技术原理

1. **文档解析** - 向量数据库存储规则文档
2. **语义理解** - AI 提取属性和规则逻辑
3. **情境匹配** - 根据玩家行动检索相关规则
4. **智能应用** - 自动选择属性和应用规则

这就是充分发挥大模型能力的正确方式！
