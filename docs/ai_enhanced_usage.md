# AI 增强功能使用指南

## 概述

重构后的 TRPG 插件充分发挥大模型能力，AI 不再只是工具调用者，而是真正的智能 KP/DM。

## 核心特性

### 1. 智能场景叙述

**传统模式：**
```
玩家: 我进入房间
AI: 你进入了房间
```

**AI 增强模式：**
```
玩家: 我进入房间
AI: 你推开吱呀作响的木门，一股霉味扑面而来。房间里光线昏暗，
    只有窗外的月光透过破碎的玻璃洒进来。墙角堆满了落满灰尘的书籍，
    书桌上的蜡烛早已燃尽...
```

AI 会自动：
- 生成沉浸式场景描述
- 根据氛围调整叙述风格
- 暗示可探索的细节

### 2. 动态剧情管理

**场景设置：**
```
玩家: 我们来到了废弃的疗养院
AI: [自动调用 set_scene("废弃疗养院", "horror")]
    [自动调用 set_tension(7)]
    
    夜幕降临，你们站在这座废弃疗养院前。建筑外墙斑驳，
    窗户像空洞的眼睛凝视着你们。风吹过，似乎能听到
    从建筑深处传来的呜咽声...
```

**紧张度自动调整：**
- 探索阶段：紧张度 1-3，鼓励玩家尝试
- 发现线索：紧张度 3-5，制造悬念
- 遭遇危险：紧张度 7-9，提高检定难度

### 3. 智能 NPC 互动

**创建 NPC：**
```
玩家: 我们遇到了旅馆老板
AI: [自动调用 create_npc("innkeeper", "老板娘玛莎", 
     "热情但略显紧张", "在这里经营旅馆20年")]
    
    一位中年妇女从柜台后站起来，脸上挤出笑容：
    "欢迎光临！这么晚了还有客人真是少见..."
    她的手指不安地搓着围裙，目光时不时瞥向楼梯。
```

**NPC 记忆系统：**
```
玩家: 我问她关于失踪案的事
AI: [检查 NPC 记忆和关系]
    
    玛莎的表情变得严肃："你们也听说了？上个月确实有个
    年轻人住进来后就再也没出现过..."
    [自动调用 npc_remember("innkeeper", "玩家询问失踪案")]
```

### 4. 情境感知检定

**智能难度调整：**
```
场景：探索阶段，紧张度低
玩家: 我搜索房间
AI: [自动降低 DC]
    请进行侦察检定（DC 12）
    
场景：高潮阶段，紧张度高  
玩家: 我搜索房间
AI: [自动提高 DC]
    时间紧迫！请进行侦察检定（DC 18）
```

**失败的艺术：**
```
玩家: 侦察检定失败
AI: 你匆忙搜索房间，虽然没找到关键线索，但注意到
    墙上的划痕似乎在指向某个方向。同时，你的动静
    惊动了楼下的守卫...
    [失败推进剧情，而非简单卡住]
```

### 5. 线索追踪

**自动记录：**
```
玩家: 我在日记中发现了一个日期
AI: [自动调用 add_clue("1924年3月15日")]
    
    日记的最后一页写着：1924年3月15日，他们来了...
    [线索已记录，后续可以关联]
```

**智能关联：**
```
玩家: 墓碑上的日期是什么？
AI: [检索已记录线索]
    墓碑上刻着：1924年3月15日
    这个日期...你在哪里见过？对了，是那本日记！
```

## 使用技巧

### 给 KP/DM 的建议

1. **信任 AI 的即兴能力**
   - 不需要事先准备所有细节
   - AI 会根据玩家行动动态生成内容

2. **使用场景管理工具**
   ```
   set_scene("神秘森林", "mysterious")
   set_tension(5)
   ```

3. **创建有深度的 NPC**
   ```
   create_npc("merchant", "商人约翰", 
              "贪婪但胆小", 
              "知道一些不该知道的秘密",
              "他目睹了仪式|他欠黑帮的钱")
   ```

### 给玩家的建议

1. **详细描述行动**
   - 好：我小心翼翼地检查门锁，寻找陷阱的痕迹
   - 差：我开门

2. **与 NPC 深度互动**
   - AI 会记住你的对话
   - NPC 态度会根据你的行为变化

3. **拥抱失败**
   - 失败检定会带来有趣的后果
   - 不是游戏结束，而是新的机会

## 配置选项

```python
# 在插件配置中添加
AI_NARRATIVE_STYLE: str = "immersive"  # 沉浸式叙述
AI_PROACTIVITY: int = 7  # AI 主动性（1-10）
AUTO_SCENE_DESCRIPTION: bool = True  # 自动场景描述
DYNAMIC_DIFFICULTY: bool = True  # 动态难度调整
```

## 技术细节

### 提示词注入层次

1. **基础身份层** - 定义 AI 是 KP/DM
2. **系统专业层** - CoC/DnD 规则知识
3. **当前状态层** - 场景、紧张度、角色信息
4. **叙事指导层** - 根据情境调整风格
5. **即兴创作层** - 应对意外的指导原则

### 数据持久化

- 剧情状态：`plot_state.{chat_key}`
- NPC 数据：`npc.{chat_key}.{npc_id}`
- 线索记录：包含在剧情状态中

## 常见问题

**Q: AI 会不会太主动，剥夺 KP 的控制权？**
A: 不会。AI 提供建议和自动化，KP 仍然掌控剧情走向。可以通过配置调整 AI 主动性。

**Q: 如何处理 AI 生成的内容与模组冲突？**
A: 使用文档上传功能，AI 会优先参考已上传的模组内容。

**Q: 失败检定真的不会卡住剧情吗？**
A: AI 遵循"失败推进剧情"原则，会生成有趣的后果而非简单阻止。

**Q: NPC 记忆会保留多久？**
A: 每个 NPC 保留最近 10 条记忆，足够维持会话连贯性。
